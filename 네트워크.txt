- 네트워크와 계층 모델
    - 컴퓨터 네트워크를 구성하는 계층들
    - 주로 애플리케이션 계층들을 사용하고 서비스 제공
    - 이슈는 어디 계층에서 발생하는지 파악 힘듬
- 웹페이지 프로토콜
    - http : 주로 애플리케이션 계층 사용
    - tcp/udp : 연결 지향/ 비연결 지향
    - ip : 인터넷 계층
- 데이터 통신의 기본 요소
    - 메시지
    - 송신자
    - 수신자
    - 전송매체
    - 프로토콜
- 네트워크 범주를 통한 구성
    - WAN: 광역통신망, 국가와 국가 또는 대륙과 대륙을 연결
    - LAN: 300M이하에 연결된 장비
    - MAN: 도시 네트워크
    - TCP/IP 프로토콜
        - 데이터를 주고 받기 위해 정한 약속 규칙
- 패킷의 흐름
    - 트랜스포트 레이어 단계
    - 전송 계층에서 패킷을 분할을 하든, 조립해서 보내든 ,원하는 패킷만 가져오든 애플리케이션 계층으로 들어옴
    - TCP or UDP
- 프로세스 vs 스레드
    - 프로세스: 하나의 형태로 실행 하는게 주요 목적
    - 스레드 --> 프로세스 하위 개념
    - 운영체제의 목적 프로세스 관리
    - 컴퓨터 안에서 실행되는것을 연산이라 함
- 소켓
    - 응용프로그램과 TCP/IP계층을 연결하기 위한 인터페이스를 소켓 인터페이스의 연결
    - 소켓의 구성
        - 데이터를 보내고 받는 부분까지 코딩
        - 서버쪽에서 먼저 준비되어야함
        - ex) 채팅 프로그램
            - 서버는 방을 만들어주는 역할
            - 프로세스의 하위 개념으로 스레드라함
            - 시스템 소켓 세팅 필요
            - 소켓 생성할 때 중간 변수를 이용하여 TCP인지 UDP인지 구분
- tcp socket API
    - bind - listen - accept -connect - send, write
- UDP socket API
    - sendto 보내는 역할
    - recvfrom 받는 역할

- 인터넷 계층의 역할
    - ip어드레스 정보를 확인후 데이터 전달
    - ipv4 = 32비트
- 어드레스 클래스
    - 8비트 - A클래스
    - 16비트- B클래스
    - 24비트- C클래스
    - 32비트-  D클래스
    - 서브넷 마스크를 통해 늘려서 쓰는 것이 가능
- 퍼블릭 ip어드레스는 각 단체에 신청해서 할당
- 인터넷 서비스 제공자가 확보한 ip 어드레스들을 빌려서 사용
- 라우팅
    - 인터넷에서 데이터가 목적지까지 제대로 전달되기 위해서는 라우터가 자신과 연결된 다른 라우터를 찾아 나가면서 최종 목적지까지 연결 되는 경로를 찾아야함
    - 라우팅 프로토콜
        - 각 라우터는 서로 누구와 연결되어 있는지에 대한 정보를 교환할 때 사용
        - BGP, OSPF, RIP  등
    - 네트워크 간의 패킷을 전달
    - 프로트콜 상태 확인으로 라우터들이 타입 번호들을 보냄
    - 네트워크 어드레스 변환(NAT)
        - 누가 사용하는지 정보 기억
        - 라우터 - 네트워크 어드레스 변환기
- 도메인
    - DNS - 기업을 나타낼 수도 있고 사람이 문자를 외우는 능력이 더 뛰어남
    - DNS는 서버 안 쪽에  등록을 하고 사용
- DHCP
    - IP 어드레스를 자동으로 할당
    - 일반적으로는 공유기에 탑재
